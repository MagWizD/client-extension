# PromptRanger — UWB Hacks the Future 2026

> Built in 48 hours at UWB Hacks the Future 2026 -- Cities & Societies track.

PromptRanger is a VSCode extension that passively detects and attributes AI-generated code contributions by serializing flagged metadata as git notes on each commit. When a developer pushes, the notes travel to GitHub where a bot can read them on incoming pull requests and generate an attribution report for the maintainer.

The tool requires no action from the developer; it watches silently in the background and attaches its findings to the normal git workflow.

---

## The Problem

Open source maintainers are drowning in AI-generated pull requests. Code that technically runs but was generated by an AI tool — pasted in without review or understanding — degrades codebases over time and burns out the people responsible for reviewing it. Existing tools like GitHub's CodeQL and Dependabot catch security vulnerabilities and broken builds, but nothing surfaces the subtler problem: contributions that pass CI but represent no real human understanding of the codebase.

---

## What Was Built

### VSCode Extension

A passive monitoring extension that:

- Listens for suspicious code insertions using `vscode.workspace.onDidChangeTextDocument`
- Flags large instant insertions (> 200 chars in a single change event) as potential AI-generated code
- Queues flagged regions in memory and persists them to disk so they survive VSCode restarts
- Detects commits via the VSCode Git API and serializes all queued flags as a JSON git note attached to the commit SHA
- Detects pushes and automatically pushes git notes to remote alongside commits via a dual mechanism — `onPush()` in the extension and a pre-push hook installed in `.git/hooks/`
- Includes a `@ranger` chat participant that logs query/response pairs from Copilot to the same git note

### Git Notes as Transport

All metadata is stored as git notes -- a native git feature that attaches arbitrary data to commit SHAs without modifying the commit itself. Notes travel to remote with a separate push.

```bash
# View the note on the latest commit
git notes show HEAD
```

```json
{
  "aiidentifierVersion": "0.0.1",
  "commit": "abc1234...",
  "generatedAt": "2026-04-25T18:00:00.000Z",
  "flagCount": 2,
  "flaggedRegions": [
    {
      "file": "src/auth.js",
      "startLine": 42,
      "endLine": 67,
      "charCount": 1204,
      "reasonFlagged": "Large instant insertion — 1204 chars in 80ms",
      "timeStamp": "2026-04-25T17:45:00.000Z"
    }
  ],
  "chatHistory": []
}
```

## Key Technical Challenges Solved

**Git notes don't push automatically**
Git notes are separate from commits and require an explicit `git push origin refs/notes/commits` to reach remote. Built a dual mechanism — a pre-push hook that appends to any existing hook without overwriting it, and an `onPush()` listener in the extension — this way notes reliably reach remote regardless of how the developer pushes.

**Commit detection firing prematurely on activation**
When the extension restarted with queued flags on disk, the state change listener would treat the existing HEAD commit as a new one and immediately attach flags to an already-pushed commit. Solved by persisting `lastCommitSha` to disk alongside the flags so the extension always knows which commit was last processed across sessions.

**Windows shell argument escaping**
Passing large JSON directly as a git command argument broke on Windows due to how the shell handles quotes. Resolved by writing note content to a temporary file first and using git's `-F` flag to read from that file instead of passing the content inline.

**VSCode extension load order**
The extension was initially crashing because it tried to access the Git API before the built-in `vscode.git` extension had finished loading. Fixed by declaring `vscode.git` as an `extensionDependency` in `package.json`.

## What Comes Next

PromptRanger was a proof of concept. The ideas explored here are being continued under the **Aletheia** project — a platform-agnostic detection engine with distributions for VSCode, JetBrains IDEs, and other editors, paired with **Themis**, a reporting engine that reads Aletheia's git notes on incoming pull requests and generates maintainer reports across GitHub, GitLab, and Bitbucket.
